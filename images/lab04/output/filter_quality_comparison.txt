Сравнение качества работы фильтров


1. Усредняющий ФНЧ 3x3

Усредняющий фильтр хорошо подавляет аддитивный гауссов шум на фотографиях и
обычных полутоновых изображениях. Например:

  flower, Gaussian var=900:
    noisy PSNR    = 18.76 dB
    low-pass PSNR = 28.19 dB

  imagesample, Gaussian var=625:
    noisy PSNR    = 20.31 dB
    low-pass PSNR = 26.83 dB

При этом фильтр заметно размывает мелкие детали и границы. Поэтому на
изображении pixel_art_invader, где важны резкие блоковые границы, PSNR
ухудшается:

  pixel_art_invader, Gaussian var=225:
    noisy PSNR    = 24.73 dB
    low-pass PSNR = 16.53 dB

Хорошо работает для фотографий, или картинок без мелких деталей, но плохо подходит для изображений с
резкими искусственными границами и пиксельной графики.

2. Усредняющий ФНЧ с порогом

Пороговый вариант заменяет пиксел на сглаженный только при достаточно большом
отклонении. Для гауссова шума на фотографиях уступает обычному ФНЧ по
PSNR, потому что часть шумовых отклонений остается без изменения:

  imagesample, Gaussian var=625:
    low-pass PSNR           = 26.83 dB
    threshold low-pass PSNR = 25.35 dB

Для импульсного шума пороговый вариант работает лучше, потому что выбросы
черного и белого цвета хорошо отделяются от нормальных пикселов:

  flower, impulse p=0.04:
    low-pass PSNR           = 28.72 dB
    threshold low-pass PSNR = 30.90 dB

  imagesample, impulse p=0.06:
    low-pass PSNR           = 24.94 dB
    threshold low-pass PSNR = 26.27 dB

Пороговый ФНЧ лучше работает для импульсного шума, но для гауссова
шума его качество зависит от выбранного порога. 
Для пиксельной графики обладает теми же недостатками, что и обычный ФНЧ.

3. Лапласиан

Лапласиан является ФВЧ-фильтром и выделяет резкие изменения яркости.

Вариант four реагирует в основном на горизонтальные и вертикальные перепады.
Вариант eight также учитывает диагональные направления, поэтому контуры обычно
становятся более плотными. При увеличении kernel-size используется больше точек
в численной аппроксимации второй производной. Это может сделать отклик более
чувствительным к протяженным перепадам, но также повышает риск усиления шума.

Файлы для визуального сравнения:

  *_laplacian_four.png
  *_laplacian_eight.png
  *_laplacian_four_5x5.png
  *_laplacian_eight_5x5.png

Тут что-то сказать получается только на глаз, видто, что скажем для розы результат
для любого варианта лапласиана неплохой (но там мало шума и границы хорошо отделимы визуально).
Однако для картинок, где много мелких деталей или штришков (imagesample, где маслом картина)
границы превращаются в кромешный шум. Похожая ситуация с видов на город из окна в bnw.

4. Лапласиан гауссиана и пересечения нулевого уровня

LoG сначала сглаживает изображение гауссовой частью, затем применяет
лапласиан. Поэтому он устойчивее к шуму.

Файлы для визуального сравнения:

  *_log_7x7_sigma*.png 

Визуально, на первый взгляд выглядит более интересно. 
Реальные границы ярче, вид из окна и мазки кистью уже не такая каша (хотя все еще не очень)
Для розы довольно ярко видны края лепесков слева, которые почти не были заметны обычным лапласианом 
(но может быть тут дело в том, как я нормировал)
А вот для пиксельной графики сработало ощутимо хуже.
Много смешения как будто (что логично, гауссов фильтр же сначала)

5. Zero-crossing

Zero-crossing строит бинарную карту границ по смене знака LoG-отклика.
Количество найденных пикселов границы зависит от структуры изображения и
автоматического порога. Например:

  imagesample_zero_crossing_log_7x7_sigma1p4.json:
    threshold   = 2.4782
    edge_pixels = 985550

Вариант с автоматической настройкой порога выделяет как границы очень много шума.
Посмотреть на ту же розу, маленькие шумные пиксели все превратились в границы.
Увеличив размер ядра до 11x11 и увеличив sigma до 2.4 (более сильное предварительное сглаживание)
получилось добиться значительного снижения шума для фото-картинок и рисунка маслом,
но пиксель арт это убило совсем.

5. Повышение резкости

Фильтр повышения резкости строится как I + amount * L, где L - лапласиан.
Это значит, что он добавляет к исходному изображению высокочастотную
составляющую. Визуально контуры становятся резче, но шум и мелкая текстура
тоже усиливаются.

Файлы для сравнения:

  *_sharpen_3x3.png
  *_sharpen_5x5.png
  *_sharpen_7x7.png

Размер ядра меняет численную аппроксимацию производной. Малый размер дает
локальное усиление ближайших границ. Больший размер учитывает более дальние
точки и может делать эффект сильнее на протяженных переходах, но на шумных
изображениях его надо применять осторожно.

"Песочный" шум с ростом ядра сильно заметно для картинки с видом на город (bnw)
Для розы, где высокочастотный "песок" на лепестках как раз дает ощущение четкости работает очень хорошо.
Для пиксельарта убивается форма.
